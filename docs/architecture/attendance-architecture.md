# Wenxin Attendance Module — Architecture Specification

**Date:** 2026-06-08
**Status:** Draft, pending review
**Depends on:** Staff Authentication (approved, implemented)
**Product Spec:** [2026-06-08-attendance-module-design.md](2026-06-08-attendance-module-design.md)

## 1. Architecture Overview

The Attendance module is a **standalone L2 module** under the People category (peer to Staff and Schedule). It adds two database tables, one new L2 route (`/attendance`), extends the existing `/staff` route with real shift data, introduces clock-in/out server actions, replaces sample data with live queries on the Home dashboard, and integrates with the existing audit logging system.

All new tables use Row Level Security matching the four-role model. All clock events and corrections are audit-logged via the existing `write_operational_audit()` trigger function.

### Module Hierarchy

```
People
  ├── Staff       (/staff/accounts, /staff/activity)
  ├── Schedule    (/staff)
  └── Attendance  (/attendance)  ← NEW (standalone L2)
```

### Principles

- **Status is derived, not stored.** Attendance status (present/late/absent/on_leave/off/pending) is computed at query time from shift assignments and clock timestamps. The database stores only facts.
- **Sessions, not daily records.** A staff member can have multiple attendance sessions per calendar day. Each session is an independent row. Clock in creates a new session; clock out closes it.
- **Server actions for clock events.** Clock in/out uses server actions (not direct browser Supabase calls) to ensure audit logging, session validation, and orphaned-session handling are consistent.
- **Existing patterns, not new ones.** Route protection uses `requireRole()` / `proxy.ts`. Data access uses `createServerSupabaseClient()` for server components and `supabase` from `@/lib/supabase/client` for client components. Audit logging reuses the existing `write_operational_audit()` trigger.

## 2. Data Model

### 2.1 `staff_shifts`

Stores the weekly shift assignment for each staff member. One row per staff member per date. Replaces the hardcoded `shifts` map in `app/staff/page.tsx`. Shared between the Schedule and Attendance modules.

```sql
create table public.staff_shifts (
  id bigint generated by default as identity primary key,
  staff_user_id uuid not null references public.staff_profiles(id) on delete cascade,
  date date not null,
  shift_type text not null check (shift_type in (
    'morning', 'full', 'afternoon', 'off', 'leave'
  )),
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now(),
  unique (staff_user_id, date)
);

create index staff_shifts_date_idx on public.staff_shifts(date);
create index staff_shifts_staff_date_idx on public.staff_shifts(staff_user_id, date);
```

**Shift types and their time windows:**

| shift_type | Label | Time Window | Short Code |
|------------|-------|-------------|------------|
| `morning` | Morning | 09:00–15:00 | 09–15 |
| `full` | Full | 10:00–20:00 | 10–20 |
| `afternoon` | Afternoon | 14:00–21:00 | 14–21 |
| `off` | Off | — | Off |
| `leave` | Leave | — | Leave |

**Rules:**
- Shift types with time windows (`morning`, `full`, `afternoon`) are considered "scheduled to work" for attendance derivation.
- `off` means the staff member is not scheduled but may still clock in (cover shift). Sessions without a linked shift are allowed.
- `leave` means the staff member is on approved leave. Clock-in is still permitted (staff may come in despite leave), but the attendance board shows `on_leave` unless they have an active session.
- `shift_start` and `shift_end` are derived from `shift_type` at query time. They are not stored as columns.
- The unique constraint on `(staff_user_id, date)` means each staff member has at most one shift assignment per day. A missing row is equivalent to `off`.

### 2.2 `attendance_sessions`

Stores clock in/out events. **Multiple rows per staff per day are allowed.** Each row is one continuous work session. Created on clock in; updated on clock out or manual correction.

```sql
create table public.attendance_sessions (
  id bigint generated by default as identity primary key,
  staff_user_id uuid not null references public.staff_profiles(id) on delete cascade,
  schedule_id bigint references public.staff_shifts(id) on delete set null,
  business_date date not null default current_date,
  clock_in timestamptz not null,
  clock_out timestamptz,
  clock_method text not null default 'app' check (clock_method in (
    'app', 'manager_manual', 'owner_correction'
  )),
  end_reason text check (end_reason in (
    'app', 'auto_closed', 'manager_manual', 'owner_correction'
  )),
  correction_note text,
  corrected_by uuid references public.staff_profiles(id),
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create index attendance_sessions_date_idx on public.attendance_sessions(business_date);
create index attendance_sessions_staff_date_idx on public.attendance_sessions(staff_user_id, business_date);
create index attendance_sessions_open_idx on public.attendance_sessions(staff_user_id)
  where clock_out is null;
```

**Column notes:**
- `schedule_id` — optional reference to a `staff_shifts` row. Set at clock-in time if a matching shift exists (same staff, same date, shift window contains current time). Null if no shift is assigned or the clock-in falls outside any assigned shift window. `on delete set null` so deleting a shift assignment does not cascade-delete attendance history.
- `business_date` — the calendar date this session belongs to. Defaults to the current date at insert time. For v1, clock-in and clock-out are assumed to be on the same calendar date.
- `clock_in` — when the session started. Always set (not null — a session record only exists because someone clocked in or a manager created it).
- `clock_out` — when the session ended. Null if the session is still open.
- `clock_method` — who recorded the clock event. `app` for self-service; `manager_manual` for Manager corrections; `owner_correction` for Owner corrections. Set at session creation and updated on correction.
- `end_reason` — how the session ended. `app` for normal self-service clock-out; `auto_closed` when a new clock-in automatically closes an orphaned open session; `manager_manual` or `owner_correction` for manual closure. Null while the session is open.
- `correction_note` — required when `clock_method` is `manager_manual` or `owner_correction` and a correction was made. Null for pure `app` sessions.
- `corrected_by` — the `staff_profiles.id` of the Manager or Owner who made the correction. Null for `app`.
- No unique constraint on `(staff_user_id, business_date)` — multiple sessions per day are explicitly supported.

**Partial index on open sessions:** The filtered index `attendance_sessions_open_idx` supports fast lookups for "does this staff member have an open session?" queries.

### 2.3 `restaurant_settings`

Stores owner-configurable restaurant-level settings. Key-value design accommodates future settings without schema changes.

```sql
create table public.restaurant_settings (
  key text primary key,
  value text not null,
  updated_at timestamptz not null default now(),
  updated_by uuid references public.staff_profiles(id)
);

-- Seed default
insert into public.restaurant_settings (key, value) values ('late_threshold_minutes', '15');
```

**Rules:**
- Only Owner can read or write settings (enforced by RLS).
- Values are stored as text. The application layer parses them.
- `late_threshold_minutes` must be an integer between 5 and 60.

## 3. Derived Values (Computed at Query Time)

These values are never stored. They are computed in server components or utility functions at read time.

### 3.1 Shift Time Window

```typescript
const SHIFT_WINDOWS: Record<string, { start: string; end: string }> = {
  morning:   { start: '09:00', end: '15:00' },
  full:      { start: '10:00', end: '20:00' },
  afternoon: { start: '14:00', end: '21:00' },
}

function getShiftWindow(shiftType: string): { start: string; end: string } | null {
  return SHIFT_WINDOWS[shiftType] ?? null
}
```

### 3.2 Match Session to Shift

At clock-in time, the system attempts to match the session to a shift assignment:

```typescript
function matchShift(
  staffUserId: string,
  businessDate: string,
  clockInTime: Date,
  shifts: StaffShift[]
): number | null {
  // Find a shift for this staff on this date whose time window contains clockInTime
  const match = shifts.find(shift => {
    if (shift.staff_user_id !== staffUserId || shift.date !== businessDate) return false
    const window = getShiftWindow(shift.shift_type)
    if (!window) return false
    const [sh, sm] = window.start.split(':').map(Number)
    const shiftStart = new Date(clockInTime)
    shiftStart.setHours(sh, sm, 0, 0)
    const [eh, em] = window.end.split(':').map(Number)
    const shiftEnd = new Date(clockInTime)
    shiftEnd.setHours(eh, em, 0, 0)
    return clockInTime >= shiftStart && clockInTime <= shiftEnd
  })
  return match?.id ?? null
}
```

### 3.3 Attendance Status (per staff, per day)

```typescript
type AttendanceStatus = 'present' | 'late' | 'absent' | 'pending' | 'on_leave' | 'off'

function deriveStatus(
  shifts: StaffShift[],              // all shifts for this staff on this date
  sessions: AttendanceSession[],     // all sessions for this staff on this date
  lateThresholdMinutes: number,
  now: Date
): AttendanceStatus {
  // No working shifts assigned
  const workingShifts = shifts.filter(s =>
    s.shift_type !== 'off' && s.shift_type !== 'leave'
  )
  const hasLeaveShift = shifts.some(s => s.shift_type === 'leave')
  const allOff = shifts.every(s => s.shift_type === 'off')

  // Has an open session → present
  if (sessions.some(s => !s.clock_out)) return 'present'

  // Has at least one closed session
  if (sessions.length > 0) {
    const hasLateSession = sessions.some(s => {
      const linkedShift = shifts.find(sh => sh.id === s.schedule_id)
      if (!linkedShift) {
        // Try to find any working shift for this date
        const anyShift = workingShifts[0]
        if (!anyShift) return false
        return isLate(s.clock_in, anyShift.shift_type, lateThresholdMinutes)
      }
      return isLate(s.clock_in, linkedShift.shift_type, lateThresholdMinutes)
    })
    return hasLateSession ? 'late' : 'present'
  }

  // No sessions at all
  if (hasLeaveShift) return 'on_leave'
  if (allOff) return 'off'

  // Has a working shift but no session
  const currentShift = getCurrentShift(workingShifts, now)
  if (!currentShift) return 'off' // between shifts

  const window = getShiftWindow(currentShift.shift_type)!
  const [eh, em] = window.end.split(':').map(Number)
  const shiftEnd = new Date(now)
  shiftEnd.setHours(eh, em, 0, 0)
  return now < shiftEnd ? 'pending' : 'absent'
}
```

### 3.4 Hours Worked (per session)

```typescript
function sessionHours(clockIn: string, clockOut: string | null): number | null {
  const end = clockOut ? new Date(clockOut) : new Date()
  const start = new Date(clockIn)
  return Math.round((end.getTime() - start.getTime()) / 3600000 * 10) / 10
}
```

### 3.5 Is Late

```typescript
function isLate(
  clockIn: string,
  shiftType: string,
  lateThresholdMinutes: number
): boolean {
  const window = getShiftWindow(shiftType)
  if (!window) return false
  const [sh, sm] = window.start.split(':').map(Number)
  const clockInTime = new Date(clockIn)
  const shiftStart = new Date(clockInTime)
  shiftStart.setHours(sh, sm, 0, 0)
  const lateThreshold = new Date(shiftStart.getTime() + lateThresholdMinutes * 60000)
  return clockInTime > lateThreshold
}
```

### 3.6 Orphaned Session Auto-Close

When a staff member clocks in while an open session already exists:

```typescript
async function handleOrphanedSession(
  staffUserId: string,
  newClockInTime: Date
): Promise<void> {
  // Find the open session
  const { data: openSession } = await supabase
    .from('attendance_sessions')
    .select('id')
    .eq('staff_user_id', staffUserId)
    .is('clock_out', null)
    .order('clock_in', { ascending: false })
    .limit(1)
    .maybeSingle()

  if (openSession) {
    // Auto-close it at the new clock-in time
    await supabase
      .from('attendance_sessions')
      .update({
        clock_out: newClockInTime.toISOString(),
        end_reason: 'auto_closed',
      })
      .eq('id', openSession.id)
  }
}
```

This logic runs inside `clockInAction()` before the new session is created.

## 6. RLS Policies

### 6.1 `staff_shifts`

```sql
-- Owner and Manager: full access
create policy staff_shifts_manage on public.staff_shifts
  for all to authenticated
  using (public.staff_role_is(array['owner', 'manager']))
  with check (public.staff_role_is(array['owner', 'manager']));

-- All authenticated: read own shifts
create policy staff_shifts_read_own on public.staff_shifts
  for select to authenticated
  using (staff_user_id = auth.uid());
```

### 6.2 `attendance_sessions`

```sql
-- Owner and Manager: full read access to all sessions
create policy attendance_sessions_read_all on public.attendance_sessions
  for select to authenticated
  using (public.staff_role_is(array['owner', 'manager']));

-- All staff: read own sessions
create policy attendance_sessions_read_own on public.attendance_sessions
  for select to authenticated
  using (staff_user_id = auth.uid());

-- Clock in: insert own session (any role, app method only)
create policy attendance_sessions_insert_own on public.attendance_sessions
  for insert to authenticated
  with check (
    staff_user_id = auth.uid()
    and clock_method = 'app'
    and corrected_by is null
    and correction_note is null
  );

-- Clock out (own session): close own open session
create policy attendance_sessions_update_own on public.attendance_sessions
  for update to authenticated
  using (
    staff_user_id = auth.uid()
    and clock_method = 'app'
  )
  with check (
    staff_user_id = auth.uid()
    and clock_method = 'app'
  );

-- Auto-close orphaned session (triggered by self clock-in)
create policy attendance_sessions_auto_close on public.attendance_sessions
  for update to authenticated
  using (
    staff_user_id = auth.uid()
    and clock_out is null
  )
  with check (
    staff_user_id = auth.uid()
    and end_reason = 'auto_closed'
  );

-- Owner and Manager: update any session (corrections)
create policy attendance_sessions_correct on public.attendance_sessions
  for update to authenticated
  using (public.staff_role_is(array['owner', 'manager']))
  with check (public.staff_role_is(array['owner', 'manager']));
```

## 7. Audit Logging

### 7.1 Operational Audit Trigger

Attach the existing `write_operational_audit()` trigger to the new tables:

```sql
-- staff_shifts
create trigger write_operational_audit
  after insert or update or delete on public.staff_shifts
  for each row execute function public.write_operational_audit();

-- attendance_sessions
create trigger write_operational_audit
  after insert or update or delete on public.attendance_sessions
  for each row execute function public.write_operational_audit();
```

### 7.2 What Gets Logged

| Event | entity_type | Action | Captured |
|-------|-------------|--------|----------|
| Clock in (new session) | `attendance_sessions` | `insert` | staff_user_id, business_date, clock_in, clock_method |
| Clock out (close session) | `attendance_sessions` | `update` | before (clock_out=null), after (clock_out set, end_reason) |
| Auto-close orphaned session | `attendance_sessions` | `update` | before (clock_out=null), after (clock_out set, end_reason='auto_closed') |
| Manager correction | `attendance_sessions` | `update` | before (original times, method), after (corrected times, method, note, corrected_by) |
| Shift assigned | `staff_shifts` | `insert` | staff_user_id, date, shift_type |
| Shift changed | `staff_shifts` | `update` | before (old shift_type), after (new shift_type) |
| Shift removed | `staff_shifts` | `delete` | the deleted row |

### 7.3 Sensitive Data

The `attendance_sessions` table contains no passwords, tokens, or secrets. The existing `PRIVATE_KEY_PATTERN` sanitizer in `lib/auth/audit.ts` is sufficient. No additional sanitization rules are needed.

### 8.3 Server Actions

```
app/attendance/actions.ts — new file

clockInAction()
  ├── requireCurrentStaff()
  ├── Check for orphaned open session → auto-close if found
  ├── Match clock-in time to a staff_shifts row (schedule_id)
  ├── INSERT into attendance_sessions (staff_user_id, schedule_id, business_date, clock_in, clock_method='app')
  ├── Audit: logged by trigger
  └── revalidatePath('/attendance')

clockOutAction()
  ├── requireCurrentStaff()
  ├── Find open session for this staff member (most recent)
  ├── Validate: at least one open session exists
  ├── UPDATE attendance_sessions SET clock_out = now(), end_reason = 'app'
  ├── Audit: logged by trigger
  └── revalidatePath('/attendance')

updateShiftAction(staffUserId, date, shiftType)
  ├── requireRole('owner', 'manager')
  ├── UPSERT into staff_shifts (staff_user_id, date, shift_type)
  ├── Audit: logged by trigger
  └── revalidatePath('/staff')

correctSessionAction(sessionId, clockIn, clockOut, note)
  ├── requireRole('owner', 'manager')
  ├── Validate: note is non-empty, clock_out > clock_in if both provided
  ├── Determine clock_method: 'manager_manual' or 'owner_correction'
  ├── UPDATE attendance_sessions SET clock_in, clock_out, clock_method, correction_note, corrected_by
  ├── Audit: logged by trigger (captures before/after)
  └── revalidatePath('/attendance')
```

## 9. Integration Points

**Changes from current state:**

1. **Quick Access grid** — Add `{ href: '/attendance', label: 'Attendance' }` card, visible to all roles (`canAccessPath` check).

## 11. Session State Machine

```
                         ┌─────────────────┐
                         │   NO OPEN SESSION │
                         └────────┬────────┘
                                  │
                                  │ clockInAction()
                                  │ • auto-closes any orphaned open session
                                  │ • matches schedule_id if shift exists
                                  │ • INSERT new session with clock_in = now
                                  ▼
                         ┌─────────────────┐
                         │   OPEN SESSION    │
                         │ clock_in = now    │
                         │ clock_out = null   │
                         │ end_reason = null  │
                         └────────┬────────┘
                                  │
                    ┌─────────────┼─────────────┐
                    │             │             │
                    ▼             ▼             ▼
           clockOutAction()  clockInAction()  correctSessionAction()
           • UPDATE           • auto-close     • Owner/Manager
             clock_out=now      previous       • UPDATE times
             end_reason=app     session        • SET clock_method
           ┌─────────────┐   • INSERT new      • SET correction_note
           │   CLOSED     │     session       ┌─────────────┐
           │   SESSION    │   ┌─────────────┐ │  CORRECTED   │
           │ end_reason=  │   │ AUTO-CLOSED  │ │  SESSION     │
           │ app          │   │ end_reason=  │ │ clock_method=│
           └─────────────┘   │ auto_closed  │ │ manager_manual│
                             └─────────────┘ │ or owner_     │
                                             │ correction   │
                                             └─────────────┘

After any close/correct path → back to NO OPEN SESSION (can start new session)
```

## 12. Summary Of Constraints

1. **Standalone L2 module at `/attendance`.** Not a sub-page of Staff. Registered in `stackRoutes.tsx` as a lazy-loaded client component.
2. **No new BottomNav tab.** Attendance is accessed from Quick Access and the Schedule page.
3. **No dashboard redesign.** Home dashboard changes are data-source swaps (sample → real) plus one new Quick Access card.
4. **Multiple sessions per day.** `attendance_sessions` has no unique constraint on `(staff_user_id, business_date)`.
5. **Orphaned sessions auto-close.** A new clock-in closes any existing open session before creating a new one.
6. **No changes to existing auth, RLS, or audit infrastructure.** The module consumes these services; it does not modify them.
7. **No changes to the four-role model.** Kitchen and Front Desk have self-service only. Manager has full view + correction. Owner has full access including settings.
8. **All visible text is English.**
9. **Existing database tables are not altered.** Only new tables are created.
10. **The existing `write_operational_audit()` trigger is reused** — no new audit functions are created.
